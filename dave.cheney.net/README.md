# Архив сайта 
[Dave Cheney](https://dave.cheney.net/) — один из самых влиятельных разработчиков языка Go (Golang), бывший Go-адвокат в Google и активный член сообщества с 2011 года. Его персональный технический блог dave.cheney.net служит важным образовательным ресурсом, где он делится глубокими инсайтами о программировании на Go, лучших практиках, производительности и инженерной культуре.

В ходе работы удалось успешно сохранить онлайн-публикацию, что подтверждается тем, что архив успешно вопроизводится на replay webpage. Общий объем содержимого страницы составил 147 мб

## ArchiveReady

Показатель архивируемости 57% является удовлетворительным, но свидетельствующим о пробленом состоянии сайта с точки зрения вем-стандартов. 
Преимущества:
- высокая когезия (67%);
- метаданные в целом присутствуют (75%);
- наличие RSS-лент и sitemap.xml.gz облегчает индексацию и архивирование контента;
- большинство ссылок валидны (98%).

Уязвимости:
- низкое соответствие веб-стандартам (33%) (все проверенные CSS-файлы содержат ошибки по валидатору W3C; HTML-документ имеет структурные ошибки)
- проблемы доступности (53%) (наличие inline JavaScript негативно влияет на доступность для ботов и вспомогательных технологий; robots.txt содержит Disallow, ограничивающий доступ к части сайта);
- 2 невалидные внешние ссылки, что снижает архивную полноту.

analyze
metadata
index
dump

Распределение данных значительно смещено в сторону документов и медиа. Всего 19 PDF файлов, но они занимают более 56% общего объёма. Это указывает на наличие крупных публикаций, отчётов или книг, которые формируют основную нагрузку.Значительный HTML-контент говорит о большом количестве страниц (статей, постов, архивов), что типично для блогов. CSS и JavaScript вместе занимают менее 0,25% от общего объёма. Это свидетельствует о лёгкой фронтенд-части и низкой зависимости от тяжёлых клиентских скриптов.

` metawarc metadata dave.cheney.net.warc.gz `

Полученные метаданные записаны в файл metadata.dave.cheney.net.txt

` metawarc index dave.cheney.net.warc.gz `

Команда оказалась необходимой для реализации dump. В результате ее работы команды на основе метаданных была создана база данных.

` metawarc dump -m "application/pdf" -o pdf `

В результате работы команды был сформирован под-архив, который включил в себя все pdf-файлы, содержащиеся в основном архиве. Результат выгрузки расположен на облаке в соотстветвующей папке архива, ссылка содержится в файле ` pdfs.md `
